# Контекст проекта Dashboard

Обновлено: 2026-05-21.

## Репозиторий и окружение

- Рабочая директория на сервере: `/home/viktor/Документы/Project/dashdoard`.
- Основная ветка: `main`.
- Remote: `https://github.com/Goriachevviktor/dashboard.git`.
- Frontend: `dashboard.html`, отдается контейнером `dashboard-web` через volume, пересборка frontend не требуется.
- Backend: FastAPI в `server/api/app/main.py`; после backend-правок нужно пересобирать `dashboard-api` через `cd server && docker compose up -d --build dashboard-api`.
- PostgreSQL: контейнер `dashboard-db`, база/пользователь `dashboard`.

## Текущее состояние функциональности

### Пространство пользователей

- Данные разделов ограничиваются видимостью пользователя: автор, исполнитель, соисполнитель или admin.
- Назначенные задачи отображаются у исполнителя, но остаются принадлежащими автору.
- Новые пользователи не получают демо-события автоматически.

### Текущие задачи и архив

- Просроченные задачи подсвечиваются.
- Вкладка `Архив` вынесена отдельно и находится в конце списка, ниже `План развития`.
- Задачи остаются в столбце `Готов`; в архив попадают после 7 дней в статусе `Готов`.
- Архив отображает задачи простым списком по наименованию, с возможностью открыть полную информацию.

### Ключевые события

- Фильтр по типу события на дорожной карте оставляет на карте только выбранный тип.
- Дорожная карта скрывает прошлые месяцы, если все события/задачи в них закрыты.
- Timeline сдвинут влево, чтобы использовать освободившееся пространство.
- Названия событий визуально оптимизированы: компактные подписи, обрезка длинных названий, tooltip/детали в выбранном событии.
- Задачи/чек-поинты из УПЦ и Плана развития с датами попадают в `Ключевые события` с типами `УПЦ` и `План развития`.
- В PWA/standalone-режиме завершенные события скрываются; в обычном browser/desktop-режиме остаются видимыми.

### УПЦ

- Задачи УПЦ видны автору и назначенным соисполнителям.
- УПЦ имеет чекбоксы выполнения на уровне задачи и контрольных точек.
- Backend хранит `ucp_tasks.done` и `ucp_checkpoints.done`.
- Состояние выполнения контрольных точек УПЦ используется для автособытий дорожной карты.

### План развития

- Добавлен `development_tasks.done` для выполнения основной задачи.
- Добавлена таблица `development_task_checkpoints` для подзадач/чек-поинтов плана развития.
- В модальном окне задачи можно добавлять подзадачи с датой и отметкой выполнения.
- В детальной области выбранной задачи можно быстро отмечать задачу и подзадачи выполненными.
- Неактивные чекбоксы из карточек задач в сетке убраны, чтобы не создавать визуальный шум.
- Чек-поинты плана развития с датами формируют события дорожной карты типа `План развития`.

### Заметки

- Текстовая область внутри растягиваемой заметки теперь занимает всю доступную высоту заметки.
- Исправлена проблема, когда при растягивании стикера текст оставался в ограниченной зоне.

## Обновление 2026-05-21: активность и сброс пароля

### Активность пользователей

- Добавлены агрегаты активности без детального журнала действий:
  - `users.last_seen_at`
  - `users.activity_count`
  - `user_activity_daily`
- `require_auth` записывает активность для авторизованных API-запросов.
- Refresh токена не считается активностью, потому что не проходит через `require_auth`.
- В админской вкладке `Пользователи` показаны:
  - активные сегодня;
  - активные за 7 дней;
  - действия за 7 дней;
  - пользователи без активности;
  - активность и последняя активность по каждому пользователю.

### Сброс пароля администратором

- Добавлен endpoint `POST /auth/users/{user_id}/reset-password`.
- Только admin может сбросить пароль пользователя.
- При сбросе генерируется временный пароль, пароль сохраняется как bcrypt-хеш.
- Активные refresh-сессии пользователя отзываются.
- В карточке пользователя появилась секция `Сброс пароля`; временный пароль показывается администратору один раз после сброса.
