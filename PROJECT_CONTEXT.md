## Docker

Основные контейнеры приложения:

- `dashboard-web` - отдает `dashboard.html`.
- `dashboard-api` - backend API.
- `dashboard-db` - база данных.

Полезные команды:

```bash
docker ps --format 'table {{.Names}}\t{{.Ports}}'
docker exec dashboard-web sh -c "grep -n 'timelineStartMonth' /srv/dashboard.html"
cd server
docker compose up -d --build dashboard-api
docker logs --tail 80 dashboard-api
```

Frontend сейчас редактируется напрямую в `dashboard.html`; для таких изменений пересборка `dashboard-web` обычно не нужна, потому что контейнер видит файл сразу. Backend-изменения требуют пересборки `dashboard-api`.

## Уже реализовано

- Разделение пространства пользователей: пользователь видит свои задачи и задачи, где он автор или исполнитель.
- Аналогичный механизм доступа применен к остальным разделам.
- Демо-события больше не создаются для новых пользователей.
- Просроченные задачи на вкладке `Текущие задачи` подсвечиваются.
- Архив задач вынесен в отдельную вкладку.
- Колонка `Готово` остается в текущих задачах; задачи попадают в архив через 7 дней после перехода в `Готово`.
- На вкладке `Архив` задачи отображаются простым списком по наименованию, но сохраняется возможность открыть полную информацию.
- В `Ключевых событиях` фильтр по типу события влияет на события, отображаемые на дорожной карте.
- Задачи и подзадачи из `УПЦ` и `План развития`, у которых есть даты, автоматически попадают в `Ключевые события` с типами `УПЦ` и `План развития`.
- Названия событий на дорожной карте компактно сокращаются и показывают полный текст через `title`.
- Прошлые месяцы скрываются с дорожной карты, если все события и задачи в них закрыты.
- Timeline дорожной карты пересчитывается от первого видимого месяца и сдвигается влево, чтобы не оставлять пустое место.

## Важные файлы

- `dashboard.html` - основной frontend, включая вкладку `Ключевые события`, timeline, задачи, архив.
- `server/api/app/main.py` - backend API и генерация событий дорожной карты из `УПЦ` и `План развития`.

## Дорожная карта: текущая логика

В `dashboard.html` внутри `EventsSection`:

- `hiddenPastMonths` определяет прошлые месяцы, которые можно скрыть.
- `visibleRoadmapEvents` оставляет только события видимого диапазона.
- `timelineStartMonth` выбирает первый видимый месяц.
- `timelineMonthSpan` задает длину видимого диапазона до конца года.
- `getMonthPct()` и `getEventPct()` позиционируют месяцы и события уже не по полным 12 месяцам, а по видимой шкале.
- Кнопка `Показать прошлое` / `Скрыть прошлое` временно возвращает полный годовой вид.

В `server/api/app/main.py`:

- `generated_roadmap_events()` добавляет события из `УПЦ` и `План развития`.
- `is_development_task_done()` определяет завершенность задач плана развития по статусу.
