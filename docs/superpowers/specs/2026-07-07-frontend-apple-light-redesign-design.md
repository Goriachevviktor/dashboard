# Редизайн фронтенда: светлый воздушный стиль с элементами Apple

**Дата:** 2026-07-07
**Статус:** согласован с пользователем (визуальные мокапы утверждены в brainstorm-сессии)

## Цель

Освежить внешний вид дашборда: перейти от тёмно-синей корпоративной темы к светлому воздушному стилю с приёмами Apple (матовое стекло, крупные заголовки, сегментированные контролы, единый системный синий). Попутно устранить найденные в аудите несостыковки: три разных синих акцента, дублированные цветовые константы, разнобой радиусов и модалок.

## Объём

**Входит:**
- Общая оболочка (десктоп): сайдбар, шапка — `frontend/src/App.jsx`
- Раздел «Текущие задачи» — `frontend/src/sections/TasksSection.jsx` (десктопные ветки)
- Раздел «Ключевые события» — `frontend/src/sections/EventsSection.jsx` (десктопные ветки)
- Раздел «Дорожные карты» — `frontend/src/sections/RoadmapsSection.jsx` (десктопные ветки)
- Раздел «Заметки» — `frontend/src/sections/SyncsSection.jsx` (десктопные ветки)
- Общие компоненты: `StatCard.jsx`, `ConfirmDialog.jsx`, `AssigneePicker.jsx` (в объёме, используемом этими разделами)
- Новый модуль дизайн-токенов — `frontend/src/theme.js`

**Не входит (осознанно):**
- Мобильная вёрстка (все `isMobile`-ветки) — остаётся визуально без изменений
- Остальные 6 разделов (Mind Map, Блок-схемы, УПЦ, АМБП, План развития, Архив, Пользователи) и экраны входа — переводятся на новый стиль позже отдельными итерациями
- Тёмная тема

## Архитектура: подход «токены + рестайл»

Создаётся `frontend/src/theme.js` — единый источник правды для палитры, типографики, радиусов, теней и стилей контролов в виде JS-констант. Секции подставляют токены в инлайновые стили (существующий идиом кодовой базы). Никакой конвертации в CSS-классы.

### Правило для мобильных веток

Мобильные render-пути должны остаться визуально без изменений. Там, где общая константа питает и десктоп, и мобильную ветку (например, `STICKER_COLORS`, `TYPE_COLOR`, `priColor`), мобильная ветка переводится на legacy-константу со старыми значениями (например, `LEGACY_STICKER_COLORS`), а основная константа обновляется для десктопа. Legacy-константы помечаются комментарием и удаляются при будущем рестайле мобильной версии.

## Дизайн-токены (theme.js)

### Типографика

- Шрифтовой стек: `-apple-system, BlinkMacSystemFont, "Inter", sans-serif` — заменяет hardcoded `"Inter"` в десктопных ветках. На Apple-устройствах рендерится SF Pro, на остальных — Inter.
- Заголовок раздела в шапке: 26px / weight 800 / letter-spacing −0.5px (вместо 18px/750).
- Микро-подписи (uppercase labels): 9–10px / weight 700 / letter-spacing 0.5–1.1px / цвет `textFaint`.

### Палитра

| Токен | Значение | Использование |
|---|---|---|
| `bg` | `linear-gradient(180deg,#fbfdff,#f5f8fd)` | фон страницы (вместо `#f0f6ff`) |
| `surface` | `#fff` | карточки, доски |
| `ink` | `#1d1d1f` | основной текст (вместо `#1e3a6e`) |
| `textSecondary` | `#6e6e73` / `#86868b` | вторичный текст |
| `textFaint` | `#a1a1a6` | подписи, плейсхолдеры |
| `accent` | `#007aff` | единственный акцент (заменяет `#2563eb`, `#3b6fe0`, `#3b82f6`) |
| `hairline` | `rgba(15,23,42,.06)` | все границы |
| `red` | `#ff3b30` (текст: `#e03131`) | опасно/высокий/дедлайн/линия «сегодня» |
| `orange` | `#ff9500` (текст: `#c77b09`) | средний/планирование |
| `green` | `#34c759` (текст: `#1d9a5b`) | готово/низкий/релиз |
| `gray` | `#8e8e93` | беклог/архив/нейтральный статус |
| `pastelBlue` | `#eaf3ff` | подложки стат-карточек |
| `pastelIndigo` | `#eef0ff` | — " — |
| `pastelGreen` | `#e9faf1` | — " — |
| `pastelAmber` | `#fff4e6` | — " — |

Правило пары «заливка/текст»: насыщенный системный цвет (`#34c759`) — для заливок (полосы Gantt, точки, чекбоксы); затемнённый вариант (`#1d9a5b`) — для текста на белом (контраст).

### Стекло (frosted glass)

- Сайдбар/шапка: `rgba(255,255,255,.72)` и `rgba(255,255,255,.6)` + `backdropFilter: blur(20px)` (+ `WebkitBackdropFilter` для Safari).
- Модалки: `rgba(255,255,255,.85)` + blur 28px, граница `rgba(255,255,255,.9)`.
- Подложка модалок: `rgba(15,23,42,.30)` + blur 8px.

### Геометрия и тени

- Радиусы: карточки 16, модалки 22, контролы/инпуты 10–11, кнопки-пилюли 999, колонки канбана 14, карточки канбана 12.
- Тень карточек (единая): `0 1px 2px rgba(15,23,42,.04), 0 8px 24px rgba(56,120,248,.07)`.
- Тень модалок: `0 32px 80px rgba(15,23,42,.18)`.
- Тень синей кнопки: `0 2px 8px rgba(0,122,255,.28)`.

### Общие стили контролов (экспортируются как готовые style-объекты/фабрики)

- **Сегментированный контрол** (iOS-стиль): контейнер `rgba(118,118,128,.12)`, радиус 9, padding 2; активный сегмент — белый, радиус 7, тень `0 1px 3px rgba(0,0,0,.08)`.
- **Пилюля-кнопка primary:** заливка `accent`, белый текст, радиус 999.
- **Пилюля-кнопка neutral:** `rgba(118,118,128,.08)`, текст `ink` (или красный текст для опасных действий).
- **Инпут:** hairline-граница, фон `rgba(255,255,255,.85)`, радиус 11.
- **z-index шкала:** токенизируется существующий разброс (120/200/300/310/320/700/1000) в именованные уровни.

## Ключевые правила стиля (утверждены на мокапах)

1. **Статусы и приоритеты — только цветным текстом, без фоновой заливки-«пилюли».** Касается: приоритетов задач, колонок в бейджах, статусов роадмапов («Активна»/«Черновик»/«Архив»), типов событий в карточке, Online/Offline в шапке.
2. **Один синий** `#007aff` во всём десктопе.
3. **Участники** — аватар (цвет из `USER_COLORS`) + имя обычным текстом, без цветных пилюль вокруг.

## Оболочка (App.jsx, десктоп)

- Сайдбар: тёмно-синий `#1e3a6e` → светлое стекло; активный пункт — подложка `#e8f2ff` + текст/иконка `#007aff`; неактивные — `#3a3a3c`; убирается белая полоска-индикатор слева.
- Логотип и аватары: градиент `linear-gradient(135deg,#38bdf8,#818cf8)`.
- Шапка: полупрозрачная с blur, крупный заголовок (Large Title), описание раздела под ним.
- Online/Offline — точка + цветной текст (зелёный/красный) без рамки и заливки; Push — нейтральная серая пилюля; разделители — hairline.
- Баннеры ошибок/оффлайна, PWA-установка, мобильная нижняя навигация — без изменений (мобильное).

## «Текущие задачи» (TasksSection, десктоп)

- **StatCard** получает новый вариант отображения: пастельная подложка в тон метрики, uppercase-подпись затемнённым тоном пастели, крупная цифра (22–24px, letter-spacing −0.5). Пастель ↔ метрика: Всего=pastelBlue, Беклог=серый `#f4f4f6`, В работе=pastelIndigo, Готово=pastelGreen, Без исполнителя=pastelAmber.
- **Тулбар доски:** фильтр приоритета → сегментированный контрол («Все» + три приоритета цветным текстом); фильтры исполнителей — чипы без рамок (активный — голубая подложка); «Добавить задачу» — синяя пилюля.
- **Колонки:** цветные заливки/рамки → нейтральные полупрозрачные поверхности (`rgba(118,118,128,.06)`, «В работе» — `rgba(0,122,255,.05)`, «Готово» — `rgba(52,199,89,.06)`); заголовок — цветная точка + тёмный текст; счётчик — цветной текст без залитого бейджа.
- **Карточки:** приоритет — цветной текст в правом верхнем углу; чипы «Автор/Исп./Соисп.» → строка «аватар + имя»; просроченные — красная hairline-рамка + срок красным; done-карточки — opacity .75. Drag & drop, разворачивание описания, AssigneePicker, архив/удаление — функционально без изменений.
- Дублированные `priColor`/`colColor` (3 определения в файле) сводятся к импорту из theme.js.

## «Ключевые события» (EventsSection, десктоп)

- **`TYPE_COLOR` → системные цвета Apple:** Совещание `#007aff`, Мероприятие `#af52de`, Релиз `#34c759`, Дедлайн `#ff3b30`, Планирование `#ff9500`, УПЦ `#30b0c7`, План развития `#5856d6`.
- Фильтры типов: чипы без рамок (точка + серый текст), активный — голубая подложка + цветной текст.
- Таймлайн: трек — hairline-серый, прогресс до «сегодня» — градиент `#38bdf8→#007aff`; метки событий — белые с hairline и лёгкой тенью, выбранная — голубая подложка + синяя рамка; маркер «Сегодня» — синий кружок с halo; текущий месяц в подписях — синий.
- Карточка события: тип — цветной текст; участники — аватар + имя; «Редактировать» — нейтральная пилюля, «Удалить» — нейтральная пилюля с красным текстом, «Добавить задачу» — синяя пилюля.
- Логика (антиколлизия меток, скрытие прошлых месяцев, автогенерируемые события) не меняется.

## «Дорожные карты» (RoadmapsSection, десктоп)

- **`BAR_COL`:** done `#34c759`, progress `#007aff`, planned `#c7c7cc` (заменяют `#22b07d`/`#3b6fe0`/`#aeb9d0`).
- **`MILESTONE_COLORS`:** `#5856d6, #007aff, #34c759, #ff9500, #ff3b30, #30b0c7, #af52de, #8e8e93`.
- **`STATUS_META`:** цветной текст без bg-заливки (Активна — зелёный текст, Черновик — оранжевый, Архив — серый).
- **Линии (утверждено в 4 итерациях):**
  - FS-зависимости — **пунктир** штрих 2 / просвет 2, толщина 1px, цвет `#a1a1a6`; наконечник стрелки сплошной, уменьшенный;
  - вертикаль «сегодня» — **сплошная красная** `#ff3b30`, 1.5px, с красной точкой сверху;
  - направляющая вехи — пунктир 1px в цвет вехи с прозрачностью ~.45.
- Сетка: hairline-линии, шапка месяцев — светло-серая полоса, текущий месяц — синий жирный; чётные строки — едва заметная подложка.
- Тулбар: «Экспорт», «Связать», «Добавить веху» — нейтральные пилюли; «Добавить задачу» — синяя пилюля.
- В легенду добавляются образцы «Зависимость» (пунктир со стрелкой) и «Сегодня» (красная черта).
- Экспорт (JSON/XLS/PNG) продолжает использовать те же константы — экспортированные артефакты автоматически получают новую палитру.
- Механика (drag/resize полос, drag вех, FS-каскад, двойной клик) не меняется.

## «Заметки» (SyncsSection, десктоп)

- Фон доски: линейная сетка → **точечная** (`radial-gradient(rgba(15,23,42,.10) 1px, transparent 1px)`, шаг 22px), фон `#fbfdff`.
- **`STICKER_COLORS` (десктоп):** акценты гармонизируются — sky `#007aff`, mint `#34c759` (текст `#1d9a5b`), amber `#ff9500` (текст `#c77b09`), violet `#af52de`, rose `#ff2d55`; пастельные surface подбираются в тон (`#eaf3ff`, `#e9faf1`, `#fff4e6`, лавандовый и розовый — аналогичной светлоты). Мобильная ветка — legacy-константа.
- Стикер: радиус 16, мягкая тень в тон акцента, спикер — uppercase цветным текстом, кнопка удаления — полупрозрачный белый кружок. Drag/resize — без изменений.
- «Создать заметку» — синяя пилюля.

## Единый паттерн модалок (все 4 раздела + ConfirmDialog)

- Подложка: `rgba(15,23,42,.30)` + blur 8px.
- Карточка: матовое стекло `rgba(255,255,255,.85)` + blur 28px, радиус 22, hairline-граница, тень `0 32px 80px rgba(15,23,42,.18)`, анимация появления сохраняется.
- Header: заголовок 18px/800/−0.4px + опциональный сабтайтл; круглая кнопка закрытия `rgba(118,118,128,.12)`.
- Enum-поля (приоритет, колонка, тип события, статус роадмапа) — сегментированные контролы с цветным текстом значений.
- Выбор цвета стикера/вехи — цветные кружки с галочкой на выбранном (вместо кнопок с подписями).
- Footer: «Отмена» — нейтральная пилюля, действие — синяя пилюля, опасное действие — красная пилюля.
- **Попутный фикс:** в `ConfirmDialog` кнопка подтверждения жёстко использует градиент `${accent}→#dc2626` даже при `tone="primary"` — при унификации кнопка становится однотонной пилюлей в цвет tone.

## Обработка ошибок и совместимость

- `backdrop-filter` дублируется как `WebkitBackdropFilter` (Safari). Для браузеров без поддержки допустимая деградация — непрозрачный белый фон (визуально приемлемо, функционально не влияет).
- Инлайновые стили не создают рантайм-ошибок; риск — визуальные регрессии. Митигируется поэтапной проверкой в браузере (см. ниже).

## Тестирование и верификация

Автотестов на стили в проекте нет; верификация:
1. `npx eslint` на изменённых файлах — чисто.
2. `npm run build` — проходит.
3. Существующие unit-тесты (`node --test src/utils/roadmapDependencies.test.js`) — не затронуты, но прогоняются.
4. Ручная проверка в браузере (headless + визуально): все 4 раздела + оболочка на десктопной ширине; мобильная ширина — убедиться, что вид «до/после» идентичен; модалки каждого раздела; drag & drop канбана; drag/resize Gantt и стикеров; экспорт роадмапа.
5. Остальные 6 разделов открываются и не сломаны (используют старые стили — допустимо по решению об объёме).
